Pi-hole MCP Server

Your home network, explained in plain English — by AI. No command line needed once it's set up.

Beginner friendly Step-by-step No coding required
What is this?

You can ask your network questions — in plain English

This tool connects your Pi-hole (the device that blocks ads on your home network) to an AI assistant. Once set up, you can type questions like a text message and get real answers about what's happening on your network.

✦ What you can do
  • 🔍  "Who is on my network right now?" — see every device and what it's doing
  • 🚨  "Was anything suspicious today?" — get an instant security check
  • 📱  "What has my kid's tablet been visiting this week?" — full browsing history by device
  • 🔒  "Block ads.doubleclick.net" — block a site on command
  • 📊  "Send me a health report on Telegram" — get a formatted summary on your phone
  • ⏰  Automatic watch mode — checks every 15 min, sends a Telegram alert if anything looks wrong
Components

What's inside — 4 parts

The system has four parts that work together. You only configure them once — after that, everything runs automatically.

🧠
LLM Layer
The AI Brain — Gemini
Powered by Google Gemini. It understands your questions, decides which tools to use, and writes back in plain English. It never sees your raw data — it asks the MCP server for information.
🔌
MCP Layer
The Protocol Bridge
Sits between the AI and your data. It exposes 29 tools the AI can call — like "get today's network stats" or "send a Telegram alert". Think of it as the AI's remote control for your network.
📡
Pi-hole Connector
Your Network Data
Connects to your Pi-hole Analytics dashboard over HTTP. Reads DNS query logs, device activity, categories, and blocked domains. Can also block or unblock domains on your command.
💬
Telegram Connector
Your Phone Notifications
Sends formatted messages to your Telegram app. Alerts, network summaries, health reports — all pushed directly to your phone. You can also send messages to the bot to trigger actions.
How they connect: Gemini (brain) → talks to → MCP bridge → which calls → Pi-hole or Telegram. Each part only talks to the part directly next to it. If one part breaks, the others still work.
Before you start

What you need to collect first

Before touching any config files, collect these 5 things. Each one takes 2–5 minutes.

  • Your Pi-hole Analytics dashboard URL
    This is where the Pi-hole analytics web page is running — usually something like http://192.168.1.10:8080. Open a browser on your home network and check if you can reach it.
  • Your Pi-hole Analytics dashboard password
    The password you chose when setting up the Pi-hole Analytics dashboard.
  • A Gemini API key (free)
    Go to aistudio.google.com/app/apikey, sign in with your Google account, and click Create API key. Copy the key — it looks like AIzaSy...
  • A Telegram bot token
    Open Telegram → search for @BotFather → send /newbot → follow the steps. BotFather will give you a token that looks like 1234567890:ABCdef...
  • Your Telegram chat ID
    Open Telegram → search for @userinfobot → send /start. It will reply with your chat ID — a number like 123456789.
Configuration

3 files to fill in — that's it

All your private information lives in 3 small files under the config/ folder. You copy a template, open the file in any text editor, and replace the placeholder text with your real values. These files are never uploaded to GitHub — they stay on your computer only.

Tip: Open these files in any plain text editor — Notepad (Windows), TextEdit (Mac), or gedit (Linux). Replace only the parts shown in orange below. Do not change anything else.

📡 File 1 — Pi-hole connector

Tells the system where your Pi-hole Analytics dashboard is and how to log in.

Step 1: In a terminal, run:

cp config/connectors/pihole.example.yaml config/connectors/pihole.yaml

Step 2: Open config/connectors/pihole.yaml and fill in your values:

config/connectors/pihole.yaml Secret — never share 
url: http://192.168.1.10:8080       # ← Your Raspberry Pi's IP address + port 8080
                                     #   Find it: open the dashboard in a browser, copy the address bar

password: your_dashboard_password   # ← The password you use to log into the dashboard
Replace the highlighted parts with your real values. Keep everything else exactly as-is.

💬 File 2 — Telegram connector

Tells the system which Telegram bot to use and where to send messages.

Step 1: In a terminal, run:

cp config/connectors/telegram.example.yaml config/connectors/telegram.yaml

Step 2: Open config/connectors/telegram.yaml and fill in your values:

config/connectors/telegram.yaml Secret — never share 
bot_token: 1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ
            # ← The token @BotFather gave you.  Paste it exactly.

default_chat_id: "123456789"
            # ← Your chat ID from @userinfobot.  Keep the quote marks.
The chat ID must stay inside quote marks — e.g. "123456789" not 123456789.

🧠 File 3 — Gemini AI

Tells the system which AI model to use and how to authenticate with Google.

Step 1: In a terminal, run:

cp config/llm/gemini.example.yaml config/llm/gemini.yaml

Step 2: Open config/llm/gemini.yaml and fill in your values:

config/llm/gemini.yaml Secret — never share 
api_key: AIzaSyABCDEFGHIJKLMNOPQRSTUVWXYZ1234
         # ← Your Gemini API key from aistudio.google.com/app/apikey

model: gemini-2.0-flash
       # ← Leave this as-is. gemini-2.0-flash is fast and free.
       #   Change to gemini-1.5-pro only if you want deeper analysis.
Leave the model line unchanged unless you specifically want a different Gemini model.

⚙️ Optional — Behaviour settings (no secrets)

These two files control how the AI behaves. They have safe defaults — you don't need to change them. But if you want to customise, here's what they do:

config/llm/prompts.yaml Safe to leave as-is 
watch_interval_minutes: 15
# How often the auto-check runs (every 15 minutes by default).
# Change to 30 if you get too many Telegram messages.

system_prompt: |
  You are a home network security assistant ...
# The AI's personality and instructions.
# Only change this if you want different AI behaviour.

watch_prompt: |
  Check Pi-hole for any security alerts ...
# What the AI checks during each automatic watch-mode tick.
# Leave as-is for standard security monitoring.
config/mcp/server.yaml Safe to leave as-is 
server_name: pihole-analytics
# The name shown in Claude Desktop (if you use it). Leave as-is.

log_level: INFO
# How much detail appears in the terminal log.
# Change to DEBUG if something isn't working and you need more info.
# Change to WARNING to see less output.
Deployment

Setting up each component — step by step

Follow these steps in order. Each step builds on the previous one.

Part A — Install the software

  1. Download the code

    Open a terminal on your Raspberry Pi (or the computer where you want to run this) and type:

    git clone https://github.com/simplyauditmanager/pihole-mcp-server
cd pihole-mcp-server
    No git? You can also download the ZIP from GitHub and unzip it.
  2. Install the required Python packages 
    pip3 install -r requirements.txt

    This installs the AI library, the MCP library, and the HTTP request library. It takes about a minute.

    If you see an error about "externally managed environment" (common on Raspberry Pi OS), use this instead:
    pip3 install --break-system-packages -r requirements.txt

Part B — Set up each component's credentials

  1. Create the Pi-hole config file 
    cp config/connectors/pihole.example.yaml config/connectors/pihole.yaml

    Open config/connectors/pihole.yaml in a text editor. Replace YOUR_PI_IP:8080 with your Pi's actual IP address and port, and replace YOUR_DASHBOARD_PASSWORD with your dashboard password.

  2. Create the Telegram config file 
    cp config/connectors/telegram.example.yaml config/connectors/telegram.yaml

    Open config/connectors/telegram.yaml. Paste in your bot token from @BotFather and your chat ID from @userinfobot.

  3. Create the Gemini config file 
    cp config/llm/gemini.example.yaml config/llm/gemini.yaml

    Open config/llm/gemini.yaml. Paste in your Gemini API key.

Part C — Test each component works

  1. Test the Pi-hole connection

    Run this single question to check the Pi-hole connector can reach your dashboard:

    python3 llm/gemini_agent.py --prompt "How many devices are on my network today?"

    You should see a response like: "Today your network has 8 active devices with 14,500 queries..."

    Connection refused? Your Pi's IP or port is wrong in pihole.yaml. Check the URL by opening it in a browser.
    401 Unauthorized? Your dashboard password is wrong in pihole.yaml.
  2. Test Telegram

    Ask the agent to send you a message:

    python3 llm/gemini_agent.py --prompt "Send me a hello message on Telegram"

    Check your Telegram app — you should receive a message from your bot within a few seconds.

    No message? Your bot token or chat ID is wrong in telegram.yaml. Also make sure you have sent at least one message to your bot first — bots cannot send the first message.
  3. Test the full system together 
    python3 llm/gemini_agent.py --prompt "Check my network for anything suspicious and send the results to Telegram"

    This tests all three components at once — Gemini thinks, Pi-hole reads the data, and Telegram delivers the result to your phone.

Part D — Run the assistant

  1. Interactive mode — ask questions in a conversation 
    python3 llm/gemini_agent.py

    Type any question and press Enter. Type quit to stop.

    You: What sites has the smart TV been visiting?
Agent: The smart TV (192.168.1.25) visited 43 streaming domains today...

You: Block ads.doubleclick.net
Agent: Done. ads.doubleclick.net is now blocked on Pi-hole.
  2. Watch mode — automatic monitoring, Telegram alerts

    This runs forever in the background, checking every 15 minutes and sending a Telegram message if anything looks wrong.

    python3 llm/gemini_agent.py --watch

    Press Ctrl+C to stop. To change how often it checks, edit config/llm/prompts.yaml → watch_interval_minutes.

  3. Make watch mode start automatically on reboot (optional but recommended)

    Create a service file so watch mode runs 24/7 without you having to do anything:

    sudo nano /etc/systemd/system/pihole-mcp-watch.service

    Paste this in (replace /home/pi with your actual home folder path):

    [Unit]
Description=Pi-hole MCP Watch Agent
After=network.target

[Service]
ExecStart=/usr/bin/python3 /home/pi/pihole-mcp-server/llm/gemini_agent.py --watch
WorkingDirectory=/home/pi/pihole-mcp-server
Restart=on-failure
User=pi

[Install]
WantedBy=multi-user.target

    Save the file (Ctrl+X, then Y, then Enter), then run:

    sudo systemctl daemon-reload
sudo systemctl enable --now pihole-mcp-watch

    Done — watch mode will now start automatically every time the Pi reboots.

    To check if it's running: sudo systemctl status pihole-mcp-watch

    To see live log output: sudo journalctl -u pihole-mcp-watch -f

Troubleshooting

Something not working?

What you seeWhat it meansHow to fix it
FileNotFoundError: pihole.yaml You haven't created the config file yet Run cp config/connectors/pihole.example.yaml config/connectors/pihole.yaml
Connection refused The dashboard isn't reachable at the URL you entered Open the dashboard URL in a browser. If it doesn't load, the IP or port is wrong in pihole.yaml
401 Unauthorized Wrong dashboard password Check password in pihole.yaml — must match dashboard.password in the dashboard's config
No Telegram message received Wrong bot token or chat ID, or you haven't messaged the bot yet Send /start to your bot first. Re-check your token and chat ID in telegram.yaml
Quota exhausted from Gemini You've hit the free API limit for the minute Wait 60 seconds and try again. Or increase watch_interval_minutes to 30
AI gives weird or unhelpful answers The system prompt may need adjusting Edit config/llm/prompts.yaml → system_prompt to give the AI clearer instructions
